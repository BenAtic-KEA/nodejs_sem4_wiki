Fetch

Fetch is used to request data from a Api/server, and returns data in JSON or XML

Fetch only requires a 'URL' to request an server and returns a Promise
There is different options you can add to the Fetch call to specify what action you intend to do. By default its a GET request

Example:

                    // Get request
fetch("URL")
.then(response => response.json())
.then((result)=> {
    console.log(result)
}
                    // Post request
fetch("URL", {
// Kind of request
    method: 'POST'
// Data type
    headers: {
      'Content-Type': 'application/json',
    },
// body
    body: JSON.stringify(data),
  })
    .then((response) => response.json())
    .then((data) => {
        console.log(data);
    })
});

Redirect

Redirection is a way to forwarding a client from requested URL to another URL.

From a SEO perspective would you always aim for Server-side redirection, so you are able to control what information you are sending to the client

There is 2 kinds of redirection:

  • Server-side redirection

    • Server side redirects will be tracked through 30x http status codes.

    • The server determines what URL the client need to be sent to and is NOT handled by the browser.

  • Client-side redirection

    • redirects is handle by the browser, and is often injected through HTML or Javascript.

    • Often used as a way to update your URL (redirect to the same url)
